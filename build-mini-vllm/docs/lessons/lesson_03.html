为什么需要这个知识 Learn

前几课的零件（矩阵乘法、attention、RoPE…）单看都是孤立的小函数。这一课把它们接成一条流水线， 就得到了「你自己的 LLM」的骨架：喂进一串 token id，吐出一串 logits。后面所有课—— Lesson 5 怎么用 logits 采样出下一个 token、 Lesson 7 怎么缓存中间的 K/V、再往后的批处理与调度—— 全都跑在这条流水线上。所以这里必须把两件事彻底吃透： 数据依次流过哪些子层，以及每一步的 shape（形状）是什么。

这一课不追求「模型聪明」
权重是用固定种子随机初始化的（没训练过），所以生成的文字是乱码——这完全正常。 我们要的不是能力，而是一条结构正确、shape 正确、逐位可复现的前向通路。真实 Llama 和它的差别只是「层数/维度大几个数量级 + 权重是训练得到的」，数据流一模一样。

先看问题：不组装会怎样 Learn

假设你只有零散的零件，想「手动」算出下一个 token 的概率，会立刻卡在三个地方：

  • token id 不是向量，没法做矩阵运算。输入是 [2, 72, 105] 这样的整数 （字节级 tokenizer：每个 UTF-8 字节就是一个 id，取值 0..255，外加 BOS/EOS/PAD 共 259 个）。 整数不能直接乘权重矩阵——必须先经 Embedding 查表变成 hidden=32 维向量。
  • attention 不知道谁在前谁在后。点积对顺序无感：把 token 顺序打乱， 结果一样。语言里顺序是命根子，所以要用 RoPE 把位置注入 Q/K。
  • 单层学不到复杂映射，深层又会数值失控。只有一层表达力太弱；堆很多层， 每层输出的尺度会漂移、梯度会消失。于是需要 RMSNorm（稳住尺度）+ 残差（让信号和梯度有直达通路）+ SwiGLU（逐位置的非线性变换）。

把这些零件按固定顺序接起来、每层重复 num_layers=2 次，最后用 LM head 把 hidden=32 投影回 vocab=259 维——每一维就是一个候选 token 的分数 （logit）。这条完整流水线，正是 TinyTextModel.forward() 干的事。

直觉与全景数据流 Learn

一句话解释
Transformer = 查表把 token 变向量 → 每层反复做两件事 （先「让每个 token 看看别人」= attention，再「让每个 token 自己想一想」= 前馈）→ 最后投影回词表算分数。

类比：一场多轮圆桌会议。每个 token 是一位与会者，手里先拿到一份自我介绍（embedding）。 每一「层」就是一轮会议：先互相听取发言（attention：每个人根据自己关心的点，去加权吸收别人说的内容）， 再各自回房间独立消化（SwiGLU 前馈：逐位置地非线性加工）。开完 2 轮， 主持人根据每位与会者的最终状态，给出「下一个词该是什么」的评分（LM head）。

类比的边界
「会议」里发言是双向的，但本模型是 causal（因果）的：位置 i 的 token 只能听位置 0..i （当前及之前）的发言，不能偷看未来——否则生成时就作弊了。RoPE 提供「谁坐在第几号位」的信息， 因果掩码则强制「只能回头看」。

下面把每个子块拆开细讲，并指到本课真实源码的具体函数。

Embedding 与字节级 tokenizer Learn

Embedding 就是一张 (vocab, hidden) = (259, 32) 的查找表：第 tid 个 token 的向量， 就是这张表的第 tid 行。代码里就一句：h = [list(self.embed[tid]) for tid in input_ids] （transformer.py forward 的第 1 步）。

字节级 tokenizer：为什么 vocab 恰好是 259
本课不用复杂分词，而是逐字节：一个 UTF-8 字节（0..255）就是一个 token id，另加 BOS/EOS/PAD 三个特殊 token，共 256+3=259。所以英文一个字母 = 1 个 token； 而中文一个字 = 3 个字节 = 3 个 token。例如「你」的 UTF-8 是 3 个字节，就查 3 次表、变成 3 行向量。 这解释了为什么 vocab_size=259、以及 LM head 最后一维为什么是 259。

RMSNorm：把每个 token 的向量拉回稳定尺度 Learn

动机（具体数字）：堆 2 层、每层两次相加（残差），向量的数值会越滚越大。若某个 token 的 hidden 向量整体是 [20, -18, 25, …] 这种大尺度，下一层的矩阵乘就可能溢出或让 softmax 饱和。 RMSNorm 在每个子块入口把向量「拉回单位尺度」，让后续计算稳定。

一句话 + 类比
RMSNorm = 把一行向量按它自己的「均方根长度」等比例缩小，再乘一个可学习权重。
类比：不管你原来声音多大，先统一调到标准音量，再按每个频段的旋钮（weight）微调。

逐符号公式（输入一行 x，长度 = hidden=32）：

ms = mean(xi2) # 均方：所有分量平方的平均 rms = ms+eps # 均方根，eps=1e-5 防止除零 yi = xirms·weighti # 逐分量：先缩放，再乘权重
  • x_i：这一行第 i 个分量；ms：平方的平均（标量）；
  • eps：数值保护项 1e-5；weight_i：可学习的逐通道缩放（本模型初始化为全 1）。
  • 注意：不减均值、不加 bias——这就是它比 LayerNorm 简单、快的原因。

小规模手算（为看清楚用 hidden=4，真实是 32；weight 全 1，eps 忽略）：

x₀x₁x₂x₃
输入 x20-24
x²40416
输出 y = x/rms0.8160-0.8161.633

算法：ms = (4+0+4+16)/4 = 6，rms = √6 ≈ 2.449，于是每个分量除以 2.449。 注意 方向没变、零还是零、正负号保留——RMSNorm 只改「整行的长度」，不改方向。

代码映射
对应 mini_vllm/model/rmsnorm.py 的 rms_norm()： ms = sum(v*v for v in row)/hidden → inv = 1/sqrt(ms+eps) → row[j]*inv*weight[j]。在 forward() 里被调用 3 处：每层的 ln1（attn 前）、ln2（MLP 前）、以及最后的 final_ln。

Q/K/V 投影与 GQA Learn

归一化后的 hn (6,32) 分别乘三个权重矩阵，得到 Query、Key、Value。关键在于 Q 和 K/V 的头数不同——这就是 GQA（Grouped-Query Attention，分组查询注意力）：

  • Wq: (32, 32) → q (6, 32) = 4 个 query head，每个 head_dim=8；
  • Wk, Wv: (32, 16) → k, v (6, 16) = 2 个 KV head，每个 head_dim=8。

做 attention 时，每个 query head 要配一个 KV head。group_size = heads/kv_heads = 4/2 = 2： 每 2 个相邻 query head 共享同一个 KV head。代码里就是 kvh = h // group_size：

GQA 为什么值得
表达力（4 个查询视角）几乎不掉，但要缓存的 K/V 少了一半——KV Cache 直接减半。 这笔账在 Lesson 7 会精确算。介于 MHA（kv=4）与 MQA（kv=1）之间。
代码映射
投影在 forward()：q=matmul(hn,wq)、k=matmul(hn,wk)、v=matmul(hn,wv)。 共享逻辑在 TinyTextModel._attention()：kvh = h // g 后，用切片 row[kvh*hd:(kvh+1)*hd] 取出这一组共享的 K/V，再交给 attention_ref.sdpa_positions() 逐 head 做因果注意力（attention 的数学细节见 Lesson 4）。

RoPE：给 Q/K 注入位置 Learn

动机：attention 的分数是 q·k 点积，它对「谁在前谁在后」完全无感—— 把序列打乱，点积不变。可是语言里「猫追狗」和「狗追猫」意思相反。RoPE（旋转位置编码） 通过把每个位置的 Q/K 向量按位置相关的角度旋转，让位置信息进入点积，而且旋转后的 q·k 只依赖两位置的相对距离——这正是我们想要的性质。

两个易错点（类比边界）
RoPE 不是「加一个位置向量」（那是老式的 sinusoidal position embedding），而是旋转； 而且它只作用于 Q 和 K，不作用于 V——因为位置只需影响「谁该注意谁」（分数）， 不需要改变「被加权的内容」（V）。

逐符号公式：把每个 head 的 head_dim=8 维两两配对成 4 对 (0,1),(2,3),(4,5),(6,7)。第 i 对按角度 θ = pos · freq_i 做 2D 旋转：

freqi = theta−2i/head_dim # theta=10000, head_dim=8 x′[2i] = x[2i]·cosθ−x[2i+1]·sinθ # 这就是把 (x[2i], x[2i+1]) 这个 2D 点 x′[2i+1] = x[2i]·sinθ+x[2i+1]·cosθ # 逆时针旋转 θ 弧度

代入本模型的数字，4 对的频率非常干净（因为 10000^0.25 = 10）：

对 i维度freq_ipos=1 的角度pos=2pos=3
0(0,1)11.02.03.0
1(2,3)0.10.10.20.3
2(4,5)0.010.010.020.03
3(6,7)0.0010.0010.0020.003

看规律：低维（对 0）转得快，高维（对 3）几乎不转。低维捕捉近距离差异、高维捕捉远距离—— 这让模型能在不同尺度上区分相对位置。

小规模手算（对 0，freq=1，位置 pos=1，角度 θ=1 弧度）：设 (x[0],x[1])=(1,0)。 cos1≈0.540, sin1≈0.841，于是 x'[0]=1·0.540-0·0.841=0.540， x'[1]=1·0.841+0·0.540=0.841。这个 2D 点从 (1,0) 被逆时针转了 1 弧度到 (0.540, 0.841)—— 长度不变（还是 1），只是转了个角。

代码映射
频率表由 rope.py 的 rope_freqs(head_dim, theta) 预计算（存在 self.freqs）。 逐 head 施加旋转的是 apply_rope_heads(x, positions, num_heads, head_dim, freqs)： 它按 positions[row_idx] 取本行的绝对位置，把每行切成 num_heads 段、每段调 apply_rope_vec 做上面那对旋转。在 forward() 里对 q 用 nh=4、 对 k 用 nkv=2 各调用一次；v 不调用。

SwiGLU：门控前馈 Learn

attention 让 token 之间交换信息；SwiGLU 则是逐位置（每个 token 独立、互不相干）的非线性加工。 它比普通「一层线性 + 激活」多一条门：两条并行投影，一条过 SiLU 激活当阀门，逐元素乘另一条。

gate = x·Wgate # (6, 32) → (6, 64) 一条投影 up = x·Wup # (6, 32) → (6, 64) 另一条投影 h = silu(gate)⊙up # 逐元素相乘（门控） (6, 64) out = h·Wdown # (6, 64) → (6, 32) 收回 hidden 其中  silu(z)= z·sigmoid(z)

注意中间维 intermediate_size=64 = 2×hidden：先扩宽到 64 做非线性，再压回 32。 「门」的含义：silu(gate) 决定 up 的每个通道能通过多少——接近 0 就几乎关掉，接近 1 就放行。

小规模手算（intermediate 用 2 演示，真实是 64）：设 gate=[1.0, -2.0]、up=[3.0, 4.0]。

通道gatesigmoidsilu(gate)up门控输出 silu·up
01.00.7310.7313.02.193（放行）
1-2.00.119-0.2384.0-0.952（几乎关掉）

通道 1 的门 gate=-2 让 silu 很小，尽管 up=4 很大，输出仍被压到 -0.952—— 这就是「门控」：模型学会按输入内容选择性放行某些特征。

代码映射
对应 mini_vllm/model/mlp.py 的 swiglu(x, w_gate, w_up, w_down)：两次 matmul 得 gate/up，silu(gate[i][j]) * up[i][j] 逐元素门控，再 matmul(h, w_down) 收回。 silu() 用了数值稳定写法（对负数走 e/(1+e)）以避免 exp 溢出。

残差连接：两处 h = h + 子块输出 Learn

每个子块（attention、SwiGLU）的输出，都不是直接替换 h，而是加回原来的 h： h = h + o、h = h + mlp（代码里的 M.add）。这两条「加法旁路」看似简单，却是深层网络能训练的关键。

一句话 + 类比
残差 = 给信号留一条直达高速路：子块只需学「在原基础上补充什么修正」，而不是重造整个表示。
类比：改稿子时用「批注/增补」而不是每次从白纸重写——原稿始终在，改动是叠加上去的。

不用残差会怎样（具体后果）：若写成 h = 子块输出（替换而非相加），随着层数增加， 前面的信息会被反复覆盖、梯度在回传时逐层衰减（梯度消失）——2 层还能凑合，几十层就基本训不动了。 残差让梯度有一条恒等直连的通路，是所有现代深层 Transformer 的标配。

最终 RMSNorm 与 LM head Learn

2 层都过完后，先做一次 final_ln（最后一次 RMSNorm 稳定尺度），再乘 lm_head: (32, 259)，把每个 token 的 hidden=32 向量投影成 vocab=259 维的分数：

hf     = rms_norm(h, final_ln)     # (6, 32)
logits = matmul(hf, lm_head)       # (6, 32) · (32, 259) = (6, 259)

输出 logits 的每一行对应一个位置，每一列对应词表里的一个候选 token—— 第 (i, t) 个数就是「位置 i 之后，下一个 token 是 t」的原始分数（未归一化，越大越可能）。

生成时只用最后一行
前向对所有 6 个位置都算了 logits，但自回归生成只关心最后一行 logits[-1]—— 即「已知这 6 个 token，下一个该是谁」。前 5 行在 prefill 阶段用于填充 KV Cache（Lesson 7）、 也可用于训练时的并行监督，但采样下一个 token 时只取最后一行。这解释了因果题里「只用最后一行 logits」的含义。

常见误解 Learn

误解为什么错
「RMSNorm 会减均值、改变向量方向」RMSNorm 只按均方根等比缩放，不减均值、不加 bias，方向和正负号都不变；减均值的是 LayerNorm。
「位置信息是给 embedding 加一个位置向量」本模型用 RoPE：旋转 Q/K 而非相加，且不碰 V；旋转后的点积只依赖相对距离。
「GQA 把注意力头从 4 压成 2，少了一半头」仍是 4 个 query head在算注意力；只是 K/V 投影少到 2 份，被每 2 个 query head 共享。省的是 KV Cache，不是注意力头。
「残差只是个可选的性能小技巧」没有残差，深层网络会梯度消失、信息被逐层覆盖，基本训不动。它是深层 Transformer 的结构性必需。
「SwiGLU 就是普通线性层 + 激活」它有两条投影（gate 和 up），gate 过 SiLU 后与 up 逐元素相乘做门控——比单条线性表达力更强。
「LM head 输出的所有行都要用来生成」生成下一个 token 只用最后一行 logits[-1]；其余行用于填 KV Cache 或训练监督。

代码映射：forward() 逐步 Learn

下面的伪代码对应 mini_vllm/model/transformer.py 里 TinyTextModel.forward()（约 107–164 行）。 右侧注释标出每步用到的真实函数与 shape：

h = [embed[tid] for tid in input_ids]        # 查表 → (6, 32)
for li, layer in enumerate(layers):
    hn = rms_norm(h, layer["ln1"], eps)      # rmsnorm.py            (6, 32)
    q  = matmul(hn, layer["wq"])             # 4 个 query head       (6, 32)
    k  = matmul(hn, layer["wk"])             # 2 个 KV head (GQA)    (6, 16)
    v  = matmul(hn, layer["wv"])             #                       (6, 16)
    q  = apply_rope_heads(q, positions, 4, 8, freqs)  # rope.py：旋转 q
    k  = apply_rope_heads(k, positions, 2, 8, freqs)  # rope.py：旋转 k（v 不转）
    attn = self._attention(q, k, v, ...)     # kvh=h//2 共享；sdpa_positions
    h  = M.add(h, matmul(attn, layer["wo"])) # 残差① (Wo 拼回 hidden)
    hn2 = rms_norm(h, layer["ln2"], eps)     # 第二个子块入口 norm
    h  = M.add(h, swiglu(hn2, w_gate, w_up, w_down))  # mlp.py + 残差②
hf     = rms_norm(h, final_ln, eps)          # 最后一次 norm         (6, 32)
logits = matmul(hf, lm_head)                 # 投影到词表            (6, 259)
本课四个焦点函数，各自在哪
  • TinyTextModel.forward()（transformer.py）：把所有子块按上面的顺序串起来，是主线。
  • rms_norm()（rmsnorm.py）：均方根归一，被调用 3 次（ln1 / ln2 / final_ln）。
  • apply_rope_heads()（rope.py）：逐 head 旋转注入位置，对 q、k 各调一次。
  • swiglu()（mlp.py）：门控前馈子块。
本课的 Build 任务就是把这条 forward 补全，让 run 3 打印出 logits.shape = (6, 259) 且相同 seed 逐位可复现。
