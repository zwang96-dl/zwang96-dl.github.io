为什么需要这个知识 Learn

Attention 是 Transformer 唯一让 token 之间交换信息的地方。没有它，第 5 个字 永远不知道第 1 个字是什么——模型就退化成一个「逐字独立预测」的东西，写不出连贯的句子。 所以这一步不是可选的优化，而是语言模型的核心机制。

更关键的是：把这五步彻底算清楚，后面全都顺理成章。本课程后半段几乎每一个工程话题， 都是在这套骨架上做优化——

  • 为什么要缓存 K/V（Lesson 7）：看懂第 5 步「输出 = 历史 value 的加权和」， 就明白历史的 K/V 会被后面每一步反复用到，值得存下来。
  • 为什么 decode 每步只算 1 个 token（Lesson 8）：新 token 只贡献一行新的 Q/K/V， 历史部分完全不变。
  • GQA / PagedAttention：都是在「谁和谁做点积、结果怎么存」上动刀，数学骨架一模一样。
本课的坐标
本课只讲单个 head、prefill（Tq==Tk）的最朴素形态，把五步数学吃透。多头、GQA、 按绝对位置的 decode 版本（sdpa_positions()）留到 Lesson 7 之后——但它们调用的都是同一套 QKᵀ → scale → mask → softmax → ×V。

先看问题：不「按相关度加权」会怎样 Learn

假设序列有 3 个 token，现在要为第 3 个位置（q₂）产出一个「融合了上文的向量」。 手上有 3 个 value 向量 V=[[10,0],[0,10],[5,5]]（可以理解成每个历史 token 携带的信息）。 怎么把它们揉成一个向量？三种做法对比：

做法权重算出的向量问题
只看自己（不看上文）[0, 0, 1][5, 5]完全丢掉前文，无法指代/承接
平均所有历史（固定权重）[⅓, ⅓, ⅓][5, 5]不相关的 token 和相关的一样重，抓不住重点
Attention（按相似度加权）[0.198, 0.401, 0.401][3.98, 6.02]—（相似的 key 权重更大，自动聚焦）

看差别：平均法给出 [5,5]——三个 value 一视同仁；attention 因为发现 q₂ 和 k₁,k₂ 更像（点积更大），就把权重倾斜过去，得到 [3.98,6.02]。 这个「让相关的 token 说话更响」的权重，就是 attention 的全部意义。那份权重从哪来？ 正是下面五步里的 QKᵀ → softmax。

一句话解释 + 直觉类比 Learn

一句话解释
每个 query 用「它和各个 key 的相似度」做一份权重（softmax 归一，和为 1）， 再用这份权重把各个 value 加权求和，得到它的输出。causal 保证只加权「当前及之前」的 value。

类比：开会时分配注意力。你（一个 query）要发言，先扫一眼在座每个人（key）—— 谁刚说的内容和你要讲的最相关，你就多听谁几分（相似度 → 权重）；然后把大家的观点按这个比重 糅进你的发言（value 加权求和）。causal 就是「只能听已经发过言的人，不能预知还没开口的人要说什么」。

类比的边界（别被带偏）
「相关程度」不是人的语义理解，而是学出来的向量点积 q·k——两个向量方向越一致， 点积越大。它没有「懂不懂意思」，只有「向量像不像」。另外，softmax 权重是连续分布而非 「选一个人」：即使最相关的 key 权重 0.5，其它 key 仍分到剩下的 0.5，一起被揉进结果。

五步精确定义 Learn

先把符号和 shape 钉死（用本课单 head 的真实数字：head 维 d = head_dim = 8， prefill 时 query 与 key 数量相等 Tq = Tk = T）：

量shape（符号）本模型具体数字（设 T=3）含义
Q(Tq, d)(3, 8)每行是一个位置的查询向量
K(Tk, d)(3, 8)每行是一个位置的键向量
V(Tk, dv)(3, 8)每行是一个位置携带的信息（本模型 dv=d=8）
out(Tq, dv)(3, 8)每个 query 的输出向量

五步就是把 Q, K, V 变成 out 的流水线，每一步的 shape 明确：

步公式shape（T=3）逐符号解释
1scores = Q·Kᵀ(3,3)scores[i][j] = Σₜ Q[i][t]·K[j][t]：第 i 个 query 与第 j 个 key 的点积（相似度）
2scaled = scores / √d(3,3)每个元素除以 √d = √8 ≈ 2.83，把数值拉回合理范围（见下一节）
3masked = mask(scaled)(3,3)把 j > i 的位置（未来）改成 -∞，其余不变
4weights = softmax(masked)(3,3)逐行做 softmax：每行变成非负、和为 1 的权重分布；-∞ 那格变 0
5out = weights·V(3,8)out[i] = Σⱼ weights[i][j]·V[j]：用第 i 行权重对所有 value 加权求和

合成一行就是经典的 attention 公式： out = softmax( (Q·Kᵀ) / √d + mask ) · V。下面把「为什么除以 √d」和「掩码到底屏蔽了什么」 两个最容易含糊的点单独讲透。

数据流示意（内联执行图）

scores (3,3)

只有 ③ 会「删信息」（把未来涂成 -∞）；其余四步都是纯算术变换。scores/weights 始终是 (Tq,Tk) 方阵， out 恢复到 value 的宽度 (Tq,dv)。

为什么要除以 √d Learn

这是最常被「照抄公式」略过、却很关键的一步。点积 q·k = Σₜ qₜ·kₜ 是把 d 个乘积加起来。 维度 d 越大，这个和的波动幅度就越大——粗略地说，若每个分量方差约为 1， 则 q·k 的方差约为 d，标准差约 √d。

本模型 d = 8，√8 ≈ 2.83。如果不缩放，scores 的典型幅度会到 ±3 甚至更大； softmax 对大数极其敏感：输入相差几个单位，输出就几乎变成 one-hot（一个位置≈1，其余≈0）。这有两个坏处：

场景softmax 输入softmax 输出后果
不缩放（幅度大）[6, 0][0.9975, 0.0025]几乎只看一个 key，梯度趋近 0，训练难；表达力退化
除以 √d（幅度回落）[2.1, 0][0.89, 0.11]分布平滑、可导，其它 key 仍有发言权
一句话
除以 √d 是把「点积随维度膨胀」这件事抵消掉，让 softmax 无论 d 是 8 还是 128 都工作在差不多的数值尺度上。偏偏是 √d（而非 d 或别的）， 正因为标准差 ∝ √d。

因果掩码到底屏蔽了什么 Learn

生成是自回归的：写第 i 个 token 时，只能依赖已经写出的 0..i，绝不能「偷看」还没生成的 i+1、i+2…… 因果掩码（causal mask）就是把 scores[i][j] 中所有 j > i 的格子设成 -∞。 为什么是 -∞？因为下一步 softmax 里要算 exp，而 exp(-∞) = 0—— 这个位置的权重被精确地压成 0，等于「看不见」。

掩码的形状是一个下三角矩阵（对角线及以下保留，上三角屏蔽）。以 T=3 为例，✓ 表示可见、 ✗ 表示被置 -∞：

query＼keyk₀k₁k₂
q₀（位置 0）✓✗✗
q₁（位置 1）✓✓✗
q₂（位置 2）✓✓✓
易错点
掩码条件是 j > i（未来），不是 j < i。写反了会变成「只能看未来、看不见过去」， 句首 q₀ 反而能看到全序列——生成时等于让模型作弊，训练/推理都会出错。 另外，本课的 causal_mask_apply() 要求 Tq == Tk（方阵）；decode 场景下 Tq=1、Tk=历史长度， 那时改用按绝对位置判断可见性的 sdpa_positions()（Lesson 7）。

小规模手算：从头算到底 Learn

为了能用笔算，这里把 head 维缩到 d = 2（真实模型是 8，只是算术更长，步骤完全一样）。 取动画里的同一组输入：

Q= [ 10 11 01 ] K= [ 10 01 11 ] V= [ 100 010 55 ] d=2

① scores = Q·Kᵀ。每格是一个 query 行与一个 key 行的点积，例如 scores[1][2] = q₁·k₂ = 1·1 + 1·1 = 2：

·k₀=[1,0]·k₁=[0,1]·k₂=[1,1]
q₀=[1,0]101
q₁=[1,1]112
q₂=[0,1]011

② scaled = scores / √2（1/√2 ≈ 0.707）。每格乘 0.707： 行 0 [0.707, 0, 0.707]，行 1 [0.707, 0.707, 1.414]，行 2 [0, 0.707, 0.707]。

③ masked：把 j>i 置 -∞（下三角保留）：

k₀k₁k₂
q₀0.707-∞-∞
q₁0.7070.707-∞
q₂00.7070.707

④ weights = 逐行 softmax。关键是数值稳定：每行先减去该行最大值再取 exp （减常数不改变 softmax 结果，却能防溢出）。-∞ 取 exp 直接得 0。逐行算：

  • 行 0 [0.707,-∞,-∞] → 只剩一项 → [1, 0, 0]（句首只能看自己）。
  • 行 1 [0.707,0.707,-∞] → 两项相等 → [0.5, 0.5, 0]。
  • 行 2 [0,0.707,0.707]：减最大值 0.707 得 [-0.707,0,0]； exp = [0.493,1,1]，和 = 2.493；除一下 → [0.198, 0.401, 0.401]。

⑤ out = weights·V。用每行权重对三个 value 加权求和：

query加权求和out
q₀1·[10,0][10, 0]
q₁0.5·[10,0] + 0.5·[0,10][5, 5]
q₂0.198·[10,0] + 0.401·[0,10] + 0.401·[5,5][3.98, 6.02]

这 3 行 out 就是最终结果，和网页动画、python3 course.py run 4 --trace 打印的数字 逐位一致。回头看 先看问题 那节：q₂ 的 [3.98,6.02] 正是「按相似度加权」而非平均得到的——手算全程没有任何魔法，就是这五步算术。

放回真实模型：shape 与内存 Learn

手算用 d=2、单 head，是为了能笔算。真实的 tiny model 里，attention 长这样：

配置项值对 attention 的影响
hidden_size32每个 token 的向量宽度
num_attention_heads4Q 被拆成 4 个 head 并行做 attention
num_kv_heads2（GQA）K/V 只有 2 份，每 2 个 Q head 共享 1 份 K/V
head_dim8单个 head 的 d，缩放因子 1/√8 ≈ 0.354
max_seq_len64scores 方阵最大 (64,64)

本课只做「1 个 head」，就是把 hidden=32 切成 4 份、每份 head_dim=8 后的其中一份。 核对一下：4 heads × 8 head_dim = 32 = hidden_size。多 head 只是把本课这套五步并行跑 4 遍 （K/V 因 GQA 只算 2 份、被复用），最后把各 head 的 out 拼回 32 维。

中间量的内存热点：scores 方阵
注意 scores/weights 的 shape 是 (Tq, Tk)——随序列长度平方增长。本模型单 head、 T=64 时是 64×64 = 4096 个数，×4 head = 16384，还不算什么；但真实模型 T 上万时， 这个 T² 的注意力矩阵会成为显存和带宽瓶颈——这正是 FlashAttention「不落地整张矩阵、 分块在线 softmax」要解决的问题（Lesson 4 只需理解五步语义，优化留到后面）。

常见误解 Learn

误解为什么错
「softmax 是选出最相关的那个 key」softmax 给的是连续分布，不是「选一个」。手算里 q₂ 的权重是 [0.198,0.401,0.401]，三个 key 都参与，只是比重不同。
「缩放只是随手加的常数，去掉也行」去掉 /√d，大 d 时 scores 幅度爆大，softmax 退化成近 one-hot、梯度趋 0。它是让 softmax 稳定工作的必要一步。
「掩码是把未来位置设成 0」设成 0，softmax 里 exp(0)=1 反而给了未来一份权重。必须设 -∞，让 exp(-∞)=0 才真正屏蔽。
「每列权重加起来是 1」是每行和为 1（每个 query 的权重分布），不是每列。softmax 沿 key 维（行内）归一。
「out 的宽度和 scores 一样」out 是 (Tq, dv)，宽度来自 value（本模型 8），不是 scores 的 (Tq, Tk)。第 5 步用权重把 value「加权求和」，把 Tk 这一维求和掉了。

代码映射 Learn

本课讲的五步一对一落在 mini_vllm/model/attention_ref.py 的 scaled_dot_product_attention() 里：

scores  = M.matmul(Q, M.transpose(K))   # ① Q·Kᵀ，复用 Lesson 2 的 matmul
scaled  = M.scale(scores, 1/sqrt(d))    # ② 缩放
masked  = causal_mask_apply(scaled)     # ③ j>i -> -inf（causal=True 时）
weights = softmax_rows(masked)          # ④ 逐行 softmax（数值稳定：先减每行最大值）
out     = M.matmul(weights, V)          # ⑤ 加权求和

三个关注函数各管一段：

函数对应步骤要点
scaled_dot_product_attention()串起 ①–⑤先校验 shape（d==dk、Tk==Tv、causal 时 Tq==Tk），再依次跑五步；return_stages=True 会把每个中间量交给 Trace/动画
causal_mask_apply()③ 掩码对方阵把 j > i 的格置 NEG_INF；这就是「动画里把未来涂成 -∞」那一幕
softmax() / softmax_rows()④ 归一softmax() 先减 max 防溢出、对 -inf 直接取 0；softmax_rows() 逐行调用它
与 decode 版本的关系
同文件里还有 sdpa_positions()：把「谁能看谁」从「j>i」换成「k_pos[j] > q_pos[i]」， 于是 Tq 可以是 1（decode 只喂新 token）、Tk 是全部历史。它和本课的 scaled_dot_product_attention() 数学等价，只是支持了 KV Cache 场景—— 这就是 Lesson 7 的伏笔。
