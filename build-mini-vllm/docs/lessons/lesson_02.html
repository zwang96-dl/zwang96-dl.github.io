为什么需要这个知识 Learn

先说结论：Transformer 里几乎没有别的运算，剥到底全是矩阵乘法（matmul）。 Attention 的打分 Q·Kᵀ、加权 weights·V，MLP 的升维降维，最后的 LM head…… 没有一个例外。所以「张量的 shape 怎么流动、每一步花多少乘加」不是可有可无的背景知识， 而是你之后读懂、调通每一层的第一语言。

拿本课程的 tiny model 走一遍就明白了。它的配置是 hidden=32、layers=2、heads=4、kv_heads=2（GQA）、head_dim=8、intermediate=64、vocab=259。 一条长度为 S 的序列穿过一层，要连着做这一串 matmul（每个都是本课的 (m,k)@(k,n)→(m,n)）：

步骤运算输入 shape输出 shape
Q 投影x @ Wq(S,32)@(32,32)(S,32)
K 投影（GQA 少一半头）x @ Wk(S,32)@(32,16)(S,16)
V 投影x @ Wv(S,32)@(32,16)(S,16)
注意力打分Q·Kᵀ（每个 head）(S,8)@(8,S)(S,S)
加权求和scores · V(S,S)@(S,8)(S,8)
输出投影@ Wo(S,32)@(32,32)(S,32)
MLP 升维@ W1(S,32)@(32,64)(S,64)
MLP 降维@ W2(S,64)@(64,32)(S,32)

两层之后再接一次 LM head：(S,32)@(32,259)→(S,259)，把每个 token 变成 259 个词表分数（logits）。

不懂 shape 会怎样？一个具体后果
光是「给一个 token 算出 259 个 logits」这一步，就要做 32 × 259 = 8288 次乘加 （每个输出 = 32 个数的点积，共 259 个输出）。只要你把某个矩阵的行列写反、或漏检一次 shape 相容，这 8288 个数里没有一个是对的——而且往往不报错，只是结果全错。 看懂 shape，就是把这类 bug 挡在第一道门外。

从标量到 Tensor：数据长什么样 Learn

一句话解释
Tensor（张量）= 带 shape 的多维数组。shape 是一串数字，从最外层到 最内层依次记下每一维有多长；本课程干脆用「嵌套 list」直接表示它。

类比：套娃 / 抽屉柜。标量是一颗糖；向量是一排糖（一个抽屉）；矩阵是一柜抽屉 （多排糖）；3D 张量是一排柜子。shape=(2,2,4) 就是「2 个柜子、每柜 2 个抽屉、每抽屉 4 颗糖」。

类比的边界
套娃只帮你记住「层层嵌套」这件事。但张量的每一层长度必须整齐（每个抽屉糖数一样多）， 否则就是「锯齿数组」，本课程的 shape() 会当场抛 ValueError——真实框架同样不允许。

逐级往上，正好对应到 LLM 里的数据：

名称shape 例本模型里的具体含义
标量 scalar()一个数，如 loss、一个注意力权重
向量 vector(32,)一个 token 的表示（hidden=32）
矩阵 matrix(S, 32)一条序列的全部 token（S 行，每行 32 维）
3D 张量(B, S, 32)一个 batch 的 B 条序列

从文字到张量，走一遍。本课程的 tokenizer 是 byte-level：每个 UTF-8 字节就是一个 token id（0..255）。中文一个字占 3 个字节，所以「你好」= 6 个字节 = 6 个 token。 每个 token id 查嵌入表得到一行长度 32 的向量，于是：

  • 「你好」→ 6 个 token → 嵌入后是一个 (6, 32) 的矩阵；
  • 把「你好」和另一条同长序列一起批处理 → (2, 6, 32) 的 3D 张量。

下面用 workload 里的真实小例子 tensor3d（shape=(2,2,4)，为了看得清把 hidden 缩到 4） 画出它的嵌套结构——读 shape 就是从最外层数到最内层：

seq0 → [1 2 3 4] (hidden=4)
seq1 → [5 6 7 8]
batch[1]（序列 1）

row-column rule 与 shape 相容 Learn

一句话解释
out[i][j] = A 的第 i 行 与 B 的第 j 列 的点积 = Σ_k A[i][k]·B[k][j]。名字就来自这里： row（A 的行）配 column（B 的列）。
out[i][j] = ∑ k=0 K−1 A[i][k] · B[k][j]

逐符号拆开公式，把每个字母对上具体维度（记法 (m,k)@(k,n)→(m,n)）：

  • i：输出的第几行，取值 0..m-1（m = A 的行数）。
  • j：输出的第几列，取值 0..n-1（n = B 的列数）。
  • k：被求和消去的维度，取值 0..K-1——它同时是 A 的列数 和 B 的行数，两者必须相等，这就是「相容」。
  • Σ_k：把 A 第 i 行、B 第 j 列对应的 K 对数相乘再全部加起来（一个点积）。

小规模手算（用动画里同一组数：A=(2,3)、B=(3,2)， 结果 (2,2)）。中间的 3 消去，外面 2 和 2 留下：

输出格子行 · 列展开成点积结果
out[0][0]A 第0行 · B 第0列1·1 + 2·0 + 3·14
out[0][1]A 第0行 · B 第1列1·0 + 2·1 + 3·15
out[1][0]A 第1行 · B 第0列4·1 + 5·0 + 6·110
out[1][1]A 第1行 · B 第1列4·0 + 5·1 + 6·111

蓝框（横着的行）和绿框（竖着的列）都恰好有 3 个数——这就是「相容」的直观含义： 行要和列一样长才能逐个配对相乘。

相容检查落到本模型：一层线性变换是 (S,32)@(32,32)——A 的列数 32 == B 的行数 32， 通过；输出 (S,32)。而 MLP 升维 (S,32)@(32,64) 通过、降维 (S,64)@(64,32) 通过。 一旦维度对不上（比如误把降维写成 (S,64)@(32,32)，64≠32），就该立刻报错。

形状不相容要当场报错，别硬算
(2,3)@(2,2) 无效，因为 A 的列数 3 ≠ B 的行数 2。本课程的 matmul() 会抛出带具体数字的 ValueError（实验里就是用 matmul(A, A) 触发它）——把错误提前暴露在源头，比让错误 shape 一路传下去省几小时 debug。

转置 transpose：为什么 Attention 要 Kᵀ Learn

先看问题。Attention 要算「每个 token 的 Q 和每个 token 的 K 有多像」。本模型每个 head 的 Q 是 (S, 8)、K 也是 (S, 8)（head_dim=8）。可 (S,8)@(S,8) 不相容（列数 8 ≠ 行数 S）——直接乘不了。

一句话解释
转置 = 行列互换：(m,n) → (n,m)，即 aᵀ[j][i] = a[i][j]。 把 K 转成 Kᵀ 是 (8, S)，于是 Q·Kᵀ = (S,8)@(8,S) → (S,S)—— 正好是「每个 token 对每个 token」的相似度打分表。

手算（用本课的 A=(2,3)）：转置后每个原来的行变成列。

A (2×3) 转置成 Aᵀ (3×2)
A (2×3)
[1 2 3]
[4 5 6]
→
Aᵀ (3×2)
[1 4]
[2 5]
[3 6]
原来的第 0 行 [1 2 3] 竖起来变成第 0 列；aᵀ[j][i]=a[i][j]，比如 aᵀ[2][0]=a[0][2]=3。

这就回答了因果题里那个「为什么要对 K 转置」——不是为了好看，而是只有转置后维度才相容， 乘出来的形状 (S,S) 才是我们想要的「token×token 打分表」。

广播（broadcasting） Learn

先看问题。线性层是 xW + b：x@W 得到 (S,32)，可 bias b 只有 (32,) 一行。同一个 bias 要加到每一个 token 上。 怎么加？

  • 笨办法一：写一层 for 循环，对 S 行各加一遍——能算，但把广播这件事藏进了循环里。
  • 笨办法二：先把 b 复制成完整的 (S,32) 再逐元素加——当 S=64 时， 白白造了 64×32=2048 个数，而 bias 本来只有 32 个。
一句话解释
广播（broadcasting）= 让一个「小」张量自动沿某一维「重复对齐」到「大」张量的形状， 再逐元素运算，但不真的复制数据。本课最常见的一种是行广播： (m,n) + (n,) → (m,n)，即 add_row_bias()。

手算：拿上面算出的 C=[[4,5],[10,11]]，加 workload 里的 bias=[10,20]。 bias 被「广播」到两行，每行都加同一个 [10,20]：

常见误解 Learn

误解为什么错
「矩阵乘法就是逐元素相乘」 逐元素相乘要求同形（那是 add/scale）。matmul 是「行·列点积」， (m,k)@(k,n)→(m,n)，形状通常会变。
「(m,k)@(k,n) 里消去的是 m 或 n」 消去的永远是中间那个共享维 k；外侧的 m、n 原样保留为输出的行数与列数。
「A@B 和 B@A 一样」 矩阵乘法不满足交换律。(2,3)@(3,2)→(2,2) 可算，反过来 (3,2)@(2,3)→(3,3) 形状就不同了；很多时候一个能算、另一个直接不相容。
「转置只是把矩阵旋转/翻转一下」 转置是严格的 aᵀ[j][i]=a[i][j] 行列互换。它改变 shape (m,n)→(n,m)， 目的是让相邻两个矩阵的维度相容（如 Q·Kᵀ）。
「读 shape 从里往外读」 shape 从最外层到最内层写：(B,S,H) 是「B 条序列、每条 S 个 token、 每 token H 维」。读反了维度含义全错。

代码导读 Learn

本课三个 Build 目标函数都在 mini_vllm/model/matrix.py 里，逐一对上前面的讲解：

函数做的事对应上文
matmul(a,b)三重循环实现 row-column rule，先查 shape 相容「row-column rule」小节
transpose(a)aᵀ[j][i]=a[i][j]，行列互换「转置 transpose」小节
add_row_bias(a,bias)行广播 (m,n)+(n,)，检查 len(bias)==n「广播」小节

matmul() 的核心就是把公式摊成三重循环——每一行代码都能对回上面的手算：

for i in range(m):            # out 的每一行（对应上文的 i）
    for j in range(n):        # out 的每一列（对应上文的 j）
        s = 0.0
        for k in range(ka):   # 沿共享维 k 累加：Σ_k a[i][k]*b[k][j]
            s += a[i][k] * b[k][j]
        out[i][j] = s

动画动作 ⟶ 对应代码：动画里「高亮 A 第 i 行、B 第 j 列并逐个累加」这个动作， 就是上面最内层的 for k 循环；每点亮一个输出格子，就对应一次 out[i][j]=s。
