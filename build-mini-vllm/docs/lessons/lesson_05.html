为什么需要这个知识 Learn

模型每走一步 forward，吐出的是 259 个 logits（本模型 vocab_size=259：256 个字节值 + BOS/EOS/PAD 三个特殊符）， 每个数是「这个 token 有多该被选中」的原始分数（可正可负、未归一）。但用户要的是一个确定的 token id。 把 259 个分数压成 1 个选择的这一步，就叫采样——它是整个推理引擎最内层、每生成一个 token 都要跑一次的操作。

为什么不能永远取最高分（argmax）？因为那样同一个 prompt 每次输出一字不差，模型只会重复它「最有把握」的话， 写长了极易陷进「好的好的好的…」式的死循环。采样的价值就是给这一步可控的随机性：既不能太死板（永远最高分）， 也不能太放飞（连明显不合理的乱码 token 都可能蹦出来）。搞懂它，才能解释「为什么同样的 prompt 每次不同」 「temperature 调高发生了什么」，也才能在 Lesson 6+ 把生成循环嵌进调度器。

本课的两条主线
① 采样：如何从 259 个 logits 里挑 1 个 token（greedy / temperature / top-k / top-p）； ② 自回归循环：挑出的 token 接回输入，一步步往下生成，直到写够 max_new_tokens 或撞上停止符 EOS。 前者对应 sampling.py，后者对应 engine/generate.py 的 generate_cached()。

先看问题：只会挑最高分会怎样 Learn

拿一个具体的小例子。假设某一步模型给 4 个候选 token 的 logits 是 A=2.0、B=1.9、C=0.1、D=-3.0。 greedy（argmax）会百分之百选 A——哪怕 B 只低 0.1、几乎一样好，也永远没机会。后果：

  • 零多样性：同一 prompt 跑 100 次，100 次输出完全相同。写故事、闲聊都会很僵。
  • 易复读：一旦进入某个高分小循环（如「好的，好的，好的」），argmax 会一路选下去停不了。
  • 丢掉几乎同样好的选项：A、B 只差 0.1，softmax 后概率约 52% vs 48%，本该五五开，argmax 却把 B 判了死刑。

另一个极端是「完全随机」：259 个 token 等概率乱选 → 输出乱码。采样要解决的正是这条光谱上的取舍： temperature 控制「随机到什么程度」，top-k / top-p 控制「允许哪些 token 参与随机」。

直觉类比 Learn

一句话解释
采样 = 先用 softmax 把 259 个原始分数变成一套「概率」，再按概率掷骰子选一个 token； temperature 决定骰子「有多偏心」，top-k / top-p 决定「骰子上刻哪几个面」。

类比：一场加权抽奖。每个 token 是一张奖券，logits 越高 = 券越多、越容易被摸到。

  • temperature = 抽奖箱的「搅拌力度」。搅得越猛（T 大），冷门券也容易被摸到（更随机）； 几乎不搅（T→0），就总摸到券最多的那张（退化成 greedy）。
  • top-k = 抽奖前先把「券最少的那些人」请出场，只留前 k 名一起抽。
  • top-p = 不固定留几人，而是从券多到券少依次点名，直到被点到的人手里的券合计占了 p（比如 90%）就停。
类比的边界（别被带偏）
「掷骰子」不代表结果不可复现。本课用确定性 RNG（splitmix64，见 sampling.py 的 _Rng）： 同一个 seed 掷出的「随机数」序列永远一样，所以 同 seed + 同 logits → 同 token。 这里的「随机」指的是概率分布的形状，不是「每次运行都变」——这对调试和复现结果至关重要。

四种采样：精确定义与形状 Learn

四种方法的输入统一是一条 logits 向量，长度 = vocab_size = 259；输出是一个 token id（0 ≤ id < 259）。 它们只是在「logits → 概率 → 选择」这条流水线上，插入不同的过滤。

方法做什么特点
greedy取分数最高（argmax）完全确定
temperaturelogits/T 再 softmaxT↑ 更随机，T↓ 更保守
top-k只在最高 k 个里采样截断长尾
top-p (nucleus)累计概率达 p 的最小集合自适应候选数

① softmax —— 把分数变成概率。逐符号拆开：

pi= exp(logiti−m) ∑jexp(logitj−m)
  • logit_i：第 i 个 token 的原始分数；m：全体 logits 的最大值——减去它不改变结果（分子分母同乘一个常数），只避免 exp(大数) 溢出。
  • 输出 p_i ∈ (0,1) 且 Σ p_i = 1。输入 shape (259,) → 输出 shape (259,)。对应 sampling.py::softmax()。

② temperature —— 缩放再 softmax：logit_i / T。

  • T > 1：分数被压扁 → 概率更平均 → 更随机；T < 1：分数被拉开 → 高分更突出 → 更保守。
  • T → 0：最高分独占几乎全部概率 → 等价 greedy。代码里直接短路：if p.temperature <= 0: return argmax(logits)（见 apply_temperature() 与 Sampler.__call__）。

③ top-k —— 只留最大的 k 个概率，其余置 0。k=0 或 k≥259 时不生效（top_k_filter()）。 关键细节：置 0 后不立刻重新归一，把归一留到最后按剩余 sum 处理。

④ top-p（nucleus）—— 按概率从高到低累加，累计首次 ≥ p 时停，只留这批。 候选个数随分布自适应：分布集中时可能只留 2 个，分布分散时可能留十几个（top_p_filter()）。

第 3 步：轮盘赌采样（roulette）。以 top-k 结果 [0.644, 0.237, 0, 0] 为例，total = 0.644+0.237 = 0.881 （注意没有先归一，直接按 total 采样）。掷出随机数 r0 = rng.random()，令 r = r0 × total， 从头累加 acc，第一个让 r ≤ acc 的 token 中选：

rng.random()r = r0×0.881累加判定选中
0.700.617acc=0.644 ≥ 0.617A（id 0）
0.900.793acc=0.644 <0.793 → +0.237=0.881 ≥0.793B（id 1）

券多的 A 覆盖了 [0, 0.644] 这段更长的区间，所以更容易被 r 落中——这正是「加权抽奖」。对应 Sampler.__call__ 末尾的循环。

自回归循环与停止符 Learn

采样只产出一个 token。要生成一整句，就得把它接回输入、再跑一步——这就是自回归。 本课看 generate_cached()（KV Cache 版，Lesson 7 细讲），它分两段：

  1. prefill（一次）：把整段 prompt 喂进 model.forward(prompt_ids, positions, cache)， 得到 shape (prompt_len, 259) 的 logits；取最后一行 logits[-1]（shape (259,)）交给 Sampler，产出第一个生成 token。
  2. decode（循环）：此后每步只喂刚生成的那 1 个 token：model.forward([tok], [cur], cache)， 同样取 logits[-1] 采样，追加进结果，位置 cur += 1，直到两个停止条件之一命中。
为什么每步都取 logits[-1]（最后一行）？
forward 会为喂进去的每个位置都输出一行 logits，但我们只关心「下一个 token」——那只由序列最后一个位置预测。 prefill 喂了 prompt_len 个位置却只用最后一行；decode 只喂 1 个位置，那一行自然也是最后一行。

两个停止条件：

  • 写够了：生成数量达到 max_new_tokens（循环上限）。
  • 撞上 EOS：采样结果 == EOS_ID。本模型 EOS_ID = 257（BOS=256 / EOS=257 / PAD=258）。命中即 break—— 这就是 run 5 输出里 top_k=5 那行「遇到 257 就停」的原因。
忘了处理 EOS 会怎样
若不判 tok == EOS_ID，模型不会「自己安静下来」——它会一直生成到 max_new_tokens 撑满， 输出末尾拖一串无意义的字节 token。停止符是让「句子有个句号」的关键。
字节级 tokenizer 的小提醒
本课 tokenizer 是 byte-level：一个 UTF-8 字节 = 一个 token id（0..255），所以中文一个字 = 3 字节 = 3 个 token。 采样是逐 字节 token 进行的，不是逐「汉字」——生成中文时可能在一个字的 3 个字节中途停下。

常见误解 Learn

误解为什么错
「temperature=0 也会有点随机」代码直接短路成 argmax，根本不掷骰子。T=0 = 完全确定的 greedy，同 prompt 永远同结果。
「top-k=1 和 greedy 不一样」top-k=1 只留概率最高的一个、其余置 0，采样必然选它 —— 结果与 argmax 等价。区别只在实现路径（走没走 softmax）。
「top-p 固定留 p×vocab 个候选」p 是累计概率阈值，不是个数比例。分布尖锐时可能只留 1~2 个，平坦时留很多个 —— 个数是自适应的。
「过滤后概率不归一，采样就错了」本实现故意不重新归一，而是让轮盘赌按剩余 total = sum(probs) 采样（r = rng()×total），结果与先归一完全等价。
「temperature 和 top-k 顺序无所谓」顺序固定：先 temperature 缩放 → softmax → 再 top-k/top-p。T 改变的是概率的形状，会直接影响哪些 token 落进 top-p 的核。
「随机采样 = 结果不可复现」用带 seed 的确定性 _Rng，同 seed 必得同序列。不可复现只会发生在误用系统随机源时。

# Sampler.__call__ 的骨架（sampling.py）
if p.temperature <= 0: return argmax(logits)     # greedy 短路
probs = softmax(apply_temperature(logits, p.temperature))
probs = top_k_filter(probs, p.top_k)
probs = top_p_filter(probs, p.top_p)
total = sum(probs)
if total <= 0: return argmax(logits)             # 全被过滤光的兜底
r = self.rng.random() * total; acc = 0.0         # 轮盘赌（不先归一）
for i, pr in enumerate(probs):
    acc += pr
    if r <= acc: return i

本节总结 & 下一步

  • logits → (temperature) → softmax → (top-k/top-p) → 采样 → 下一个 token。
