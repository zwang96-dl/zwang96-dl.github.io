为什么需要这个知识 Learn

神经网络里没有「图片」这种东西——它只会做矩阵乘法。所以一张 JPG／PNG 必须先被翻译成一块 形状固定、数值规整的浮点张量，模型才看得懂。这一步叫图像预处理（image preprocessing）， 是整条多模态主线的入口：图片先在这里变成张量，下一课（Lesson 21）才能把它切成 patch_size=8、grid=2×2 的 4 个 visual token，再和文本 token 拼在一起喂进模型。 入口错了，后面全错，所以这一课要把「一张图到底怎么变成一串数」彻底讲透。

先看问题：直接把原图喂给模型会怎样 Learn

本课视觉塔的输入是写死的：image_size=16，即它只接受一张 16×16×3 的图（3 是 RGB 三个颜色通道）。可现实里用户上传的图什么尺寸都有。 假设来了三张图：

用户的原图形状 (H, W, 3)像素值直接喂进去会怎样
一张小图标(8, 8, 3)整数 0..255只有 192 个数，喂不满 768 个位置 → 形状对不上，报错
一张手机照片(3000, 4000, 3)整数 0..2553600 万个数，权重矩阵是按 16×16 设计的 → 形状对不上，报错
同一张照片(3000, 4000, 3)整数 0..255就算形状对，0..255 这么大的整数会让第一层输出爆到几千 → 训练/数值不稳

两个硬问题浮出水面：①尺寸不统一（模型要固定形状），②数值不规整 （uint8 的 0..255 太大、且是整数）。预处理就是专门解决这两件事——把任意 (H, W, 3) 的整数图，统一成 (16, 16, 3) 的、数值落在 [-1, 1] 附近的浮点张量。

直觉类比 Learn

一句话解释
图像预处理 = 把「五花八门的原图」统一裁成模型的标准证件照： 尺寸固定（resize）、颜色刻度统一（normalize）、并按模型习惯的顺序摆放数字（channel layout）。

类比：办证件照。照相馆不管你本人多高、穿什么，都会把你拍成同一规格 （比如 16×16）、同一底色、同一亮度范围的照片——因为后台系统只认这一种规格。 模型就是那个「只认标准规格」的后台系统。

类比的边界（别被带偏）
证件照会尽量「拍得好看」，但预处理不追求好看，只追求一致与稳定： 它可能把图拉变形、把颜色压成 [-1,1] 的抽象数值——人眼看着别扭，模型却正好需要。 另外，训练时用什么 mean/std、什么 image_size，推理时必须一模一样， 否则等于给模型看了一张「没见过规格」的照片，结果会明显变差。

精确定义：预处理四步流水线 Learn

本课 TinyImageProcessor 把原图变成张量，走的是下面四步。每一步只做一件事， 形状／dtype 的变化都用本模型的真实数字标出（image_size=16，mean=std=0.5）：

步骤做什么形状变化dtype
① 原图用户输入，RGB 三通道(H, W, 3) 任意uint8 整数 0..255
② resize最近邻缩放到固定边长(H, W, 3) → (16, 16, 3)uint8 整数 0..255
③ normalize逐通道 (x/255 − mean)/std(16, 16, 3) 不变浮点，落在 [-1, 1]
④ layout重排成 channel-first(16,16,3) → (3, 16, 16)浮点

normalize 公式逐符号拆解：

x_normi,j,c = xi,j,c 255 − meanc stdc
  • x[i][j][c]：第 i 行、第 j 列、第 c 个通道（c=0/1/2 即 R/G/B）的原始像素，整数 0..255。
  • / 255：先把 0..255 压到 0..1（这一步就已经从整数变成浮点了）。
  • − mean[c]：减去该通道的均值，把数据「挪到 0 附近」（本课 mean=0.5，即减 0.5）。
  • / std[c]：除以该通道的标准差，把数据「拉到统一尺度」（本课 std=0.5，即除 0.5，相当于 ×2）。
本课的巧合：mean=std=0.5 恰好是「[0,255] → [-1,1]」的线性映射
代入 mean=std=0.5：x_norm = (x/255 − 0.5) / 0.5 = x/255·2 − 1。 于是像素 0 → −1、255 → +1、正中间的 127.5 → 0。 整张图的数值被稳稳压进 [-1, 1]——第一层再乘权重也不会爆。

小规模手算：一张 2×2 小图走完全程 Learn

真实是 16×16 太大，这里用一张 2×2 的小图、并假设 image_size=4（放大）来把每一步算清楚。 原图（(H,W,3)=(2,2,3)，只写出来方便看）：

所以行的取法是 [0, 0, 1, 1]，列同理。「最近邻」就是这么朴素：不做插值、不算平均， 直接抄最近那个源像素，于是放大时每个像素被复制成一块。缩小时则会丢弃没被选中的像素。

第 ③ 步 normalize（mean=std=0.5）。只看 R 通道的几个代表值，套 x_norm = x/255·2 − 1：

注意：只有 resize 改形状里的 H/W，只有 normalize 改 dtype 与数值范围， 只有 layout 改轴的顺序——每步职责单一，出错时好定位。

channel-last 与 channel-first：同样的数，不同的摆法 Learn

normalize 之后有 16×16×3 = 768 个浮点数，它们的值一个都不变， 变的只是「按什么顺序把它们排进张量的轴里」。两种主流约定：

  • channel-last (H, W, 3)：先定位到一个像素 (i,j)，那个像素的 R/G/B 三个数挨在一起。图片文件、Pillow、多数图像库用这种。
  • channel-first (3, H, W)：先按通道分成 R 平面、G 平面、B 平面三张「单色图」叠起来。PyTorch 卷积和多数视觉模型用这种。

用一张 2×2 图（4 个像素，记作 P00 P01 P10 P11，每个含 r/g/b）看两种摆法：

channel-last (H,W,3)：按像素分组
P00:[r,g,b]
P01:[r,g,b]
P10:[r,g,b]
P11:[r,g,b]
channel-first (3,H,W)：按颜色分平面
R 平面
[r00 r01 r10 r11]
G 平面
[g00 g01 g10 g11]
B 平面
[b00 b01 b10 b11]
怎么记轴的顺序
形状元组从左到右 = 「先按哪个轴分大块，再按哪个轴分小块」。 (3,16,16) = 先分 3 个通道大块，每块里再是 16×16 的空间网格； (16,16,3) = 先铺 16×16 的空间网格，每格里再放 3 个通道。同一批数，只是嵌套顺序不同。

dtype：为什么必须从 uint8 变成浮点 Learn

dtype（数据类型）决定「一个数怎么存、能表示什么」。原图是 uint8：无符号 8 位整数， 只能表示 0..255 的整数，每个数占 1 字节。但模型的权重是浮点、梯度是小数， 算 (x/255 − 0.5)/0.5 得到的 −1.0 / 0.6 这类值，整数根本存不下。

uint8（原图）浮点（预处理后）
能表示的值整数 0..255带小数，如 −1.0、0.004、0.6
典型范围0 .. 255本课约 [-1, 1]
用途紧凑存原始像素参与矩阵乘法 / 反向传播
顺序不能反
normalize 里的 /255 一定要在浮点下算。若先按整数除，128 / 255 会被截断成 0，整张图直接塌成常数——这是新手最常踩的坑。本课代码用 / 255.0（带小数点） 强制走浮点除法，正是为了避开它。

常见误解 Learn

误解为什么错
「resize 会把图片变清晰／补出细节」恰恰相反。最近邻 resize 只会复制或丢弃已有像素：放大是复制成块（有锯齿），缩小是直接扔掉没选中的像素（丢信息），从不「凭空补细节」。
「normalize 只是把值缩小，可有可无」它是把数据移到 0 附近、拉到统一尺度，让第一层输出不爆、梯度稳定。而且推理必须用和训练相同的 mean/std，否则相当于喂了没见过规格的图，结果明显变差。
「channel-first 和 channel-last 是两份不同的数据」是同一批 768 个数，值一模一样，只是排进张量的轴顺序不同。喂给模型时必须匹配模型期望的那一种，否则通道会被当成空间维、算出乱码。
「保持 uint8 更省内存，没必要转浮点」模型的乘加运算需要小数。若不转浮点，x/255 这类除法会被整数截断（128/255→0），整张图塌成常数——省的那点内存换来的是完全错误的输入。

代码映射 Learn

上面四步全部落在 mini_vllm/multimodal/image_processor.py 的 TinyImageProcessor.preprocess() 里，一一对应：

def preprocess(self, img):
    s = self.image_size                       # = 16，模型固定输入边长
    resized = self._resize_nearest(img, s)    # ② resize：最近邻 (H,W,3) → (16,16,3)
    # ③ normalize：逐通道 (x/255 − mean)/std，注意 /255.0 强制浮点
    hwc = [[[ (resized[i][j][c] / 255.0 - self.mean[c]) / self.std[c]
              for c in range(3)] for j in range(s)] for i in range(s)]   # channel-last (16,16,3)
    # ④ layout：把最内层的通道轴提到最外层
    chw = [[[hwc[i][j][c] for j in range(s)] for i in range(s)] for c in range(3)]  # (3,16,16)
    return ProcessedImage(size=s, pixels_norm_hwc=hwc, pixels_norm_chw=chw, meta={...})
  • _resize_nearest 里的 src_i = min(h−1, int(i*h/size)) 就是手算表里那列取整——纯整数下标运算，无插值。
  • normalize 那行三重推导式对应公式 (x/255 − mean[c])/std[c]，/ 255.0 的小数点即上文「dtype 不能反」的关键。
  • hwc／chw 两个变量就是本课两种布局；preprocess 两种都返回，方便你在动画和 run 20 里对照同一批数的不同摆法。
